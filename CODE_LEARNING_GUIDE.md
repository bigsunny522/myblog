# コード・学習ガイド (Programming Learning Guide)

このサイトがどのような技術で作られているか、プログラミング初学者向けに解説します。
このガイドを読みながらコードを見ることで、Webアプリケーションの仕組みを学ぶことができます。

## 1. 使用している技術（Tech Stack）

このサイトは、現代のWeb開発で最も人気のある技術の組み合わせで作られています。

- **Next.js (App Router)**: Reactというライブラリをベースにしたフレームワークです。「ページ」や「ルーティング（URLの仕組み）」を簡単に作れます。
- **TypeScript**: JavaScriptに「型（Type）」の概念を加えた言語です。プログラムのミスを事前に防ぎやすく、コードが読みやすくなります。
- **Tailwind CSS**: デザインを作成するためのCSSフレームワークです。`className="text-red-500"` のようにクラス名を書くだけでスタイルを適用できます。
- **Framer Motion**: アニメーションを実装するためのライブラリです。要素がフワッと表示される動きなどはこれを使っています。

## 2. フォルダ構成の解説

主要なフォルダとファイルの役割を説明します。

```
my-terminal-blog/
├── app/                  # ページそのものを作る場所（Next.jsの心臓部）
│   ├── layout.tsx        # サイト全体の「外枠」（ヘッダーやフッターなど全ページ共通部分）
│   ├── page.tsx          # トップページの中身
│   ├── about/page.tsx    # 「/about」というURLで表示されるページ
│   └── globals.css       # サイト全体に適用されるCSS
│
├── components/           # 再利用可能な「部品」を入れる場所
│   ├── Header.tsx        # ヘッダーの部品
│   ├── Hero.tsx          # トップページの大きな画像エリア
│   ├── BlogList.tsx      # 記事リストを表示する部品
│   └── GearCard.tsx      # ガジェットカードの部品
│
├── content/              # 記事データ置き場
│   └── posts/            # 記事のファイル（.mdx）が入っています
│
└── lib/                  # 便利機能（ユーティリティ）やデータ定義
    └── gear.ts           # My Gearのデータが定義されています
```

## 3. コードの読み方（学習のヒント）

### コンポーネント（部品）の仕組み
React（Next.js）では、画面を「コンポーネント」という部品に分けて作ります。
例えば `components/GearCard.tsx` を見てみましょう。

```tsx
// props（親から受け取るデータ）の型定義
interface GearCardProps {
  item: GearItem; // どんなデータを受け取るかを決めています
}

// コンポーネント本体
export function GearCard({ item }: GearCardProps) {
  return (
    // HTMLのような書き方（JSX）で見た目を作ります
    <div className="...">
      <img src={item.image} ... />
      <h3>{item.name}</h3>
    </div>
  );
}
```
このように、「データを受け取って、HTML（のようなもの）を返す関数」がReactコンポーネントの基本です。

### データの流れ
トップページ（`app/page.tsx`）では、他のコンポーネントを組み合わせて画面を作っています。

1. `Hero` コンポーネントを配置
2. `BlogList` コンポーネントに記事データを渡して配置
3. `GearSection` コンポーネントを配置

まるでレゴブロックのように部品を組み立ててWebサイトが作られていることを意識してコードを読んでみてください。

## 4. 次のステップ
もし自分で機能を改造したい場合は、以下のような簡単な変更から始めてみましょう。

1. **色の変更**: `tailwind.config.ts` や各ファイルの `text-primary` などを変更してみる。
2. **文字の変更**: 各 `.tsx` ファイル内の日本語テキストを書き換えてみる。
3. **新しい部品**: ボタンを追加してみたり、新しいセクションを作ってみる。

Happy Coding!
