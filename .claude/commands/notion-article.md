# Notion記事執筆ガイド & テンプレート生成

このコマンドはNotionでブログ記事（主にレビュー記事）を執筆する際のガイドと、Notionブロック構成テンプレートを提供します。

引数があれば内容に合わせてテンプレートをカスタマイズしてください: $ARGUMENTS

---

## Notionデータベースプロパティ（必須）

記事ごとにNotionデータベースで以下を設定してください：

| プロパティ | 種別 | 説明 |
|---|---|---|
| Title | タイトル | 記事タイトル |
| Slug | テキスト | URLスラッグ（英数字・ハイフン）例: `grapht-omni-plus` |
| Subtitle | テキスト | 記事サブタイトル（任意） |
| Excerpt | テキスト | 記事の概要（SNSシェア・SEO用） |
| Date | 日付 | 公開日 |
| Category | セレクト | カテゴリ（例: レビュー） |
| Tags | マルチセレクト | タグ（複数可） |
| Published | チェックボックス | ON=公開、OFF=下書き（URLで直接アクセスは可能） |
| Recommended | チェックボックス | トップページでおすすめ表示するか |
| ページカバー | Notionカバー | サムネイル画像（`cover.jpg`として自動保存） |

## カスタムデザインコンポーネント 変換ルール（全対応）

### 1. 購入リンクカード `<BuyLinks>`

Notionブロックの並び順が重要です：

```
[H3見出し]  購入リンク
[画像ブロック]  商品画像（キャプションに「main.jpg」などファイル名を指定）
[段落]  商品名（1段落目 = タイトルとして扱われる）
[段落]  商品の説明文（2段落目 = サブテキストとして扱われる）
[箇条書き]  Amazon（リンク付き）
[箇条書き]  楽天市場（リンク付き）
[箇条書き]  公式サイト（リンク付き）
```

**リンクの種別は自動判定：**
- URLに `amazon` / `amzn` を含む → Amazonスタイル（オレンジ）
- URLに `rakuten` を含む → 楽天スタイル（赤）
- それ以外 → 公式サイトスタイル（ニュートラル）

**画像キャプションのルール：**
- `main.jpg` のようにファイル名形式で書くとそのファイル名で保存される
- キャプションがなければURLのハッシュ値で自動命名
- 保存先は `public/images/posts/<slug>/` に自動作成

---

### 2. スペックテーブル `<Specs>`

Notionの **2列テーブル** を使うと自動変換されます：

```
| 項目 | 値 |
|---|---|
| 製品名 | GRAPHT Omni Plus |
| 価格 | 6,980円 |
| 接続方式 | USB-C / 2.4GHz / Bluetooth 5.3 |
```

- ヘッダー行（「項目」「値」など）はスキップされる
- 左列 = ラベル、右列 = 値として表示

---

### 3. レビューサマリー `<ReviewSummary>` / `<ReviewPoints>` / `<ReviewPoint>`

H3見出し「GOOD」と「気になる点」のセットで自動変換されます：

```
[H3]  GOOD
[箇条書き]  **タイトル** — 説明文
[箇条書き]  **タイトル** — 説明文

[H3]  気になる点
[箇条書き]  **タイトル** — 説明文
[箇条書き]  **タイトル** — 説明文
```

**重要なルール：**
- 箇条書きのフォーマットは必ず `**太字タイトル** — 説明文`（— はダッシュ）
- 「GOOD」と「気になる点」はこの順番で隣接して配置する
- 「気になる点」H3の直前に「GOOD」セクションがないと単独で出力される

---

### 4. 特徴ポイントカード `<FeaturePoint>`

Notionの **コールアウトブロック** で数字絵文字をアイコンに指定すると、POINT カードに変換されます：

```
[コールアウト, アイコン: 1️⃣]  ホールエフェクトセンサースティック
  [段落]  スティックの入力検知に磁気を利用した...
  [画像]  （キャプションにファイル名を指定）
```

**数字アイコンの書き方（どちらでも可）：**
- 絵文字: 1️⃣  2️⃣  3️⃣  4️⃣  5️⃣  …（Notion のアイコンピッカーで選択）
- 数字1文字: `1` `2` `3` …

コールアウトの本文テキスト = POINTタイトル、子ブロック（段落・画像）= カード内容として表示されます。
テキストと画像は横並び（デスクトップ）/ 縦並び（モバイル）で自動レイアウトされます。

---

### 5. 2列・多列画像グリッド `<ImageGrid>`

Notionの **カラムレイアウト（2列・3列）** を使うと自動変換されます：

```
[2列カラムレイアウト]
  [左カラム]  [画像ブロック]（キャプション: front.jpg）
              [テキスト「全体像」]
  [右カラム]  [画像ブロック]（キャプション: grip.jpg）
              [テキスト「グリップの様子」]
```

Notion の「/columns」または「Turn into 2 columns」でカラムを作成してください。
3列・4列も同様に対応しています。

---

### 6. テキストハイライト `<Text bg="...">`

Notionのテキスト背景色が自動変換されます：

| Notion 背景色 | 表示色 |
|---|---|
| 青の背景 | `#7babff`（青マーカー、デフォルト） |
| 黄色の背景 | `#ffd600`（黄マーカー） |
| 緑の背景 | `#4ade80`（緑マーカー） |
| 赤の背景 | `#f87171`（赤マーカー） |
| 紫の背景 | `#c084fc`（紫マーカー） |

テキスト色（背景でなく文字色）も `<Text color="...">` として変換されます。

---

## Notionだけでは書けない要素

以下のみMDXで直接記述が必要です：

| 要素 | 補足 |
|---|---|
| 脚注・注釈（[1][2]スタイル） | HTML + idリンクで記述 |
| 画像と文章の任意の比率レイアウト | FeaturePoint 内では 50/50 自動になる |

## 画像管理ルール

- **カバー画像**: Notionページのカバーに設定 → `public/images/posts/<slug>/cover.jpg` に自動保存
- **本文画像**: Notionの画像ブロックに追加 → `public/images/posts/<slug>/` に自動保存
- **ファイル名指定**: 画像ブロックのキャプションに `main.jpg` / `front.jpg` などを記入
- **ファイル名未指定**: URLのハッシュ値で自動命名（再現性なし）
- **キャッシュ**: 一度ダウンロードされたファイルは再ダウンロードされない（ファイルを差し替えたい場合は手動削除）

## チェックリスト

記事を公開する前に確認してください：

- [ ] SlugがURLとして適切（英数字・ハイフンのみ、スペースなし）
- [ ] Excerptが記入されている（SNS・OGP用）
- [ ] Dateが設定されている
- [ ] Categoryが設定されている
- [ ] ページカバー画像が設定されている
- [ ] 「購入リンク」セクションの箇条書きに正しいリンクが設定されている
- [ ] GOODセクションの箇条書きが `**タイトル** — 説明` 形式になっている
- [ ] Published = ON にして公開
